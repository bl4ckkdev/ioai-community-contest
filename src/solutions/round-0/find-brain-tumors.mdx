## Problem Overview

Find Brain Tumors task involves a semi-supervised image classification problem, where participants must classify brain CT scan images into four classes (no tumor and 3 tumor types) using a limited training dataset. The training set contains only 40 labeled samples (0.02% of the full dataset), while the vast majority of data remains unlabeled. The objectve is to develop a method that can make good use of small labeled set and still generalize well, producing accurate predictions on the unseen test set.

#### Important

* **No manual labeling of unlabeled set**
* **We can't use any other pretrained model except ResNet18**

## What is SSL (Semi-Supervised Learning) ?

Semi-Supervised Learning is a type of machine learning where a model is trained on small labeled set along with larger pool of unlabeled samples. This approach is useful when labeled data is rare, but unlabeled data is easy to find.

There are many approaches in SSL, but four main methods stand out as valuable and fit for AI Olympiad problems:

## Semi-Supervised Learning Methods
### Pseudo-labeling
The model generates labels for the unlabeled set based on its current predictions learned from small labeled set. These pseudo-labeled samples are then added to the training set, allowing model to iteratively improve.

![Pseudo Labeling Image](/assets/pseudo_labeling.png)

```python
    ## Training Code
    for images_u in unlabeled_loader:
        with torch.no_grad():
            pseudo_labels = torch.argmax(F.softmax(model(images_u), dim=1), dim=1)
        preds_u = model(images_u)
        loss_pseudo = F.cross_entropy(preds_u, pseudo_labels)
```

Intuitively, the method works because predictions the model is confident about serve as reliable self-generated labels, guiding further learning.

Pseudo-labeling forms the foundation of most semi-supervised learning methods.

### Self Training with Noisy Student
Inspired from pseudo-labeling, it works as follows:

* A teacher model is trained on small labeled set
* Teacher generates pseudo labeles for unlabeled data
* Student model, specifically larger than teacher in size, is trained on both labeled and pseudo-labeled data
* During student training, noise is added to both inputs and model (Augmentation for inputs, dropout and stochastic depth for model). Note: Stochastich depth is regularization method, which randomly skips subset of layers during training. 

![Noisy Student Image](/assets/noisy_student.png)

Interesting insight: Student outperforms teacher model, even when their size is same. This highlights regularization effects in the training phase. [Noisy Student paper](https://arxiv.org/pdf/1911.04252)

### FixMatch
Fixmatch combines pseudo-labeling and consistency regularization. Model generates pseudo-labels for weakly augmented image and then enforces that predictions on strongly-augmented versions match these pseudo-labels.

Main reason why this method outperforms most of the other SSL methods is that FixMatch itself focuses solely on high-confidence predictions. It masks every prediction except >95% confidence ones, to consistently make model improve. Low confidence samples are likely incorrect, introducing noise to model if used as labels.

Let's check out code snippet to intuitively understand how it works. First of all lets create weak and strong augmentations

```python
weak_augment = T.Compose([
    T.Resize((224, 224)),         
    T.RandomHorizontalFlip(),     
    T.ColorJitter(brightness=0.1, contrast=0.1, saturation=0.1), 
    T.ToTensor(),
])

strong_augment = T.Compose([
    T.Resize((224, 224)),
    T.RandomHorizontalFlip(),
    T.RandomRotation(20),  
    T.ColorJitter(brightness=0.4, contrast=0.4, saturation=0.4, hue=0.1),
    T.RandomAffine(degrees=0, translate=(0.1,0.1), scale=(0.8,1.2), shear=10),
    T.RandomGrayscale(p=0.1), 
    T.ToTensor(),
    T.Normalize(mean=[0.485, 0.456, 0.406],
                std=[0.229, 0.224, 0.225])
])
```

Now in training phase, we have to:
* Mask low-confidence predictions to filter out potentially incorrect pseudo-labels.
* Enforce consistency by ensuring that the model’s predictions on strongly augmented images match the pseudo-labels from their weakly augmented counterparts.

```python
    weak_aug = weak_augment(images_u)
    strong_aug = strong_augment(images_u)
    with torch.no_grad():
        pseudo_labels = torch.argmax(F.softmax(model(weak_aug), dim=1), dim=1)
    preds_strong = model(strong_aug)
    mask = (torch.max(F.softmax(model(weak_aug), dim=1), dim=1).values > 0.95)
    loss_fixmatch = (F.cross_entropy(preds_strong, pseudo_labels, reduction='none') * mask).mean()

```
