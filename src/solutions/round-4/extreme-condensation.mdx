## Problem Overview

Extreme Condensation is a dataset distillation task, where participants must encode the knowledge of an entire MNIST dataset into just one synthethic training example. The goal is to construct a single 28×28 grayscale image along with a 10-dimensional soft label vector representing the digit classes (0–9).

## What is Dataset distillation

We all know about model distillation: a large, powerful teacher network transfers its knowledge into a smaller student network.

Dataset Distillation flips this idea in surprisingly interesting way. Instead of compressing model, we compress the data itself. Unlike standard dataset distillation problems, here you need to distill only ONE synthetic image and ONE synthetic label.

## Dataset Distillation Methods:

### Gradient Matching

The goal is to create a small synthetic dataset $S$ such that when a model is trained on $S$, its gradient updates resemble what it would have gotten if trained on the full real dataset $T$.

Rather than matching losses or output predictions, this method directly matches the gradients with respect to the network parameters (b):

* Compute gradients from the full dataset.  
* Compute gradients from the synthetic dataset.  
* Optimize synthetic data so that these gradients are as similar as possible.

```python
def gradient_matching_loss(snap_idx, x_syn, y_syn):
    params = {n: p.clone().requires_grad_(True) for n, p in param_snapshots[snap_idx].items()}
    loss = soft_cross_entropy(functional_call(ref_model, {**base_buffers, **params}, (x_syn,)), y_syn)
    syn_grads = torch.autograd.grad(loss, list(params.values()), create_graph=True)
    return sum((F.cosine_similarity(sg.flatten(), cached_gradients[snap_idx][n].flatten(), 0).neg() + 1) * (10 if "fc" in n else 1)
               for sg, (n, _) in zip(syn_grads, params.items()))
```
TL;DR of code:
<small>
1. Copy snapshot parameters.<br />
2. Compute loss on synthetic data.<br />
3. Compute gradients w.r.t parameters.<br />
4. Compare each gradient to <strong>cached real gradient</strong> using <strong>cosine similarity</strong>.<br />
5. Weight the FC layer higher because it affects class predictions most.<br />
6. <strong>Output:</strong> Single loss value; smaller = synthetic gradients match real ones better.
</small>

In our official solution, zeroth-order random search was used to directly maximize accuracy on a small training subset. Other than that, multiple restarts were done to choose best initialization by validation score.

### Matching Training Trajectories

* Pre‑trains an expert model on the real dataset and stores its parameter snapshots (weights over training steps).
* Uses those stored sequences of weights as targets.
![Condensation](/assets/condensation_1.jpg)

* Optimizes the synthetic data so that training a model on it creates a parameter trajectory similar to the real expert’s trajectory.
* This is done by backpropagating through multiple training steps of the synthetic training process and minimizing the difference between the student model’s parameters and those of the expert at corresponding points.
![Condensation](/assets/condensation_2.png)

```python
def train_and_eval(pixel_values, label_dist, x_test, y_test):
    net = ConvNet().to(device); net.load_state_dict(base_state_dict)
    optimizer = torch.optim.AdamW(net.parameters(), lr=0.01)
    x_syn = pixel_values.view(1,1,IMG_SIZE,IMG_SIZE).to(device)
    y_syn = label_dist.view(1,NUM_CLASSES).to(device)
    for _ in range(50): optimizer.zero_grad(); soft_cross_entropy(net(x_syn), y_syn).backward(); optimizer.step()
    return (net(x_test.to(device)).argmax(1) == y_test.to(device)).float().mean().item()
```
TL;DR of code:
<small>
1. Initialize a fresh ConvNet and load the base weights.<br />
2. Prepare the synthetic image (`pixel_values`) and label (`label_dist`) tensors.<br />
3. Train the network for 50 steps using AdamW on the single synthetic example.<br />
4. Compute soft cross-entropy loss and backpropagate each step.<br />
5. Update network parameters with optimizer step.<br />
6. Evaluate the trained network on the test set.<br />
7. <strong>Output:</strong> Classification accuracy of the network.
</small>

### Difference between them

**Gradient Matching** focuses on one-step gradients, optimizing synthetic data to produce similar parameter updates as the real dataset at a single step, which makes it simple but limited to local learning signals. **Trajectory Matching**, in contrast, aligns entire sequence of parameter snapshots, guiding the model to follow the expert’s full training path.

### Evaluation makes task harder

For the evaluation of solutions, the model is retrained using **AdamW** for exactly 50 steps, which makes the task harder compared to simple SGD. Each AdamW step depends on previous steps momentum and adaptive learning rates, so the effect of a single synthetic example accumulates in a complex way.
